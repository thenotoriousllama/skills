## 1. Feature Framing Philosophy

### The Rule

Never describe what the product does. Describe what the user gains.

Wrong: "Our AI analyzes your photo using machine learning algorithms."
Right: "{{FEATURE_FRAMING_RIGHT}}"

> Example: "Upload a photo. See your colors. Understand why some things work and others do not."

### The Three Questions

For every feature, answer:
1. What does the user understand after using this that they did not before?
2. What decision can they now make with confidence?
3. What frustration does this eliminate?

## 4. Feature Copy Templates

### Short Feature Description (50-75 words)

```
[WHAT IT REVEALS]
"{{SHORT_FEATURE_REVEAL}}"

[HOW] (1 sentence)
"{{SHORT_FEATURE_HOW}}"

[OUTCOME]
"{{SHORT_FEATURE_OUTCOME}}"
```

### Long Feature Description (100-150 words)

```
[HOOK]
Pattern: Validation Through Specificity
"{{LONG_FEATURE_HOOK}}"

[EXPLANATION]
What the feature does, framed as understanding gained.
"{{LONG_FEATURE_EXPLANATION}}"

[OUTCOME]
What changes in the user's life.
"{{LONG_FEATURE_OUTCOME}}"
```

### Feature Comparison (Entry vs. Premium)

```
[ENTRY TIER]
"{{ENTRY_FEATURE_COPY}}"

[PREMIUM TIER]
"{{PREMIUM_FEATURE_COPY}}"

[BRIDGE]
"{{FEATURE_BRIDGE_COPY}}"
> Example: "The Snapshot shows you your colors. The full system shows you everything they mean."
```
